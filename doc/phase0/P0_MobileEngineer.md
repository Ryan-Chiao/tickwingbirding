## T0-1 参与 PRD 评审（03-10 ~ 03-12）

**提前阅读范围：**
- 重点阅读 PRD v2.2 的 **第 5 章（功能需求）** — 了解移动端需要实现的功能范围
- 重点阅读 PRD v2.2 的 **第 7 章（多端策略）** — 了解移动端的能力边界和平台差异
- 重点阅读 PRD v2.2 的 **第 6 章（非功能需求）** — 关注移动端性能指标（首帧 < 2.5s、崩溃率 < 0.1%）

**评审会上需要关注的：**
- 移动端功能范围是否清晰？哪些功能 Web 有但移动端不需要？
- React Native 是否能满足所有交互需求？（视频播放器、推送通知、设备绑定）
- Phase 6 的移动端开发周期（约 10 周）是否合理？

设计师产出 Design Token 和线框图后，你需要从移动端角度 Review：

**你需要确认的内容：**
- 移动端 UI 组件是否有对应的 React Native 实现方案？

- iOS 和 Android 平台差异是否在设计中有所考虑？

**产出：** Review 意见反馈给设计师

### Phase 6 预习建议（如有空闲时间）

Phase 0-5 期间你没有开发任务，但建议利用这段时间：
- React Native 项目脚手架搭建（Expo vs bare workflow）
- 视频播放方案调研（react-native-video / expo-av）
- 推送通知方案调研（APNs + FCM，react-native-push-notification）
- 本地存储方案调研（MMKV / AsyncStorage）
- 与 Web 端共享逻辑的可行性评估（shared-types 包的使用）
