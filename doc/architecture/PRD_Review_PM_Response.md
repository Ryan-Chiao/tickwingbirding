## 一、逐项处理结果

### 数据模型

| 编号 | 问题 | 处理结果 | 说明 |
|------|------|---------|------|
| DM-01 | 缺少 refresh_tokens 表 | ✅ **已采纳** | v2.3 DDL 已新增 `refresh_tokens` 表，采用你建议的表结构（token_hash + device_info + revoked_at）。**PM 决策：Refresh Token 存储选 DB 表**，理由：需支持多设备管理和吊销审计，纯 Redis 故障时全量用户需重新登录风险过高。请在 ADR-003 中以此为准 |
| DM-02 | bind_code 无过期时间 | ✅ **已采纳** | v2.3 DDL 已在 feeders 表新增 `bind_code_expires_at` 字段。**PM 决策：有效期 15 分钟** |

| DM-04 | 鸟种名称无规范化 | ✅ **接受，列为待决策** | Phase 0 不改 DDL，Phase 3 前引入 `bird_species` 字典表。已记入待决策项 |
| DM-05 | 通知表缺少生命周期 | ✅ **已采纳** | v2.3 DDL 已在 notifications 表新增 `archived_at` 字段 + 清理索引。**PM 确认：90 天归档策略** |
| DM-06 | recorded_at 索引 | ✅ **已采纳** | v2.3 DDL 已新增 `idx_videos_recorded` 条件索引 |
| DM-07 | system_feeder | ✅ **已确认** | 维持原决策，请在数据初始化 seed 中包含此系统账号 |
